## Pillar Summary

| # | Pillar | Key fields |
|---|--------|------------|
| 1 | **General Overview** | App name*, business purpose, users, priority, RTO, RPO, compliance, risks |
| 2 | **Architecture** | Tech stack, current/target env, architecture descriptions, diagram uploads (PNG/JPG) |
| 3 | **Data Management** | Data volume, DB types, current/target DBs, retention, migration notes |
| 4 | **BC & DR** | DR strategy, backup, failover, DR testing |
| 5 | **Cost** | Current cost, migration budget, constraints, licensing |
| 6 | **Security** | Auth type, encryption (rest/transit), PII, compliance frameworks |
| 7 | **Project & Timeline** | PM, timeline, team size, dependencies, integrations, go-live |

*Only Application name is required. All other fields optional.*
